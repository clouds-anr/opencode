description: Maintain architecture and sequence documentation with PlantUML as code.
---

## Use This Skill When
- Code changes affect module interactions or runtime flow
- New architecture decisions need diagrams
- Existing diagrams drift from implementation

## Diagram Standards
- Keep one concern per diagram
- Use stable naming and aliases
- Include legend/notes for assumptions
- Prefer readable layouts over complete exhaustiveness

## Required Outputs
- Updated `.puml` sources
- Brief textual explanation of what changed and why
- Cross-reference to impacted modules and APIs
